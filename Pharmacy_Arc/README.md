## Security & Resilience (Cycle 20 Audit)

### CSRF Protection

All state-changing endpoints are protected by Flask-WTF `CSRFProtect`. The frontend fetches a token from `/api/csrf-token` and attaches it as an `X-CSRFToken` header on every POST/PUT/DELETE. Two routes are explicitly exempt:

- `/api/login` -- unauthenticated by definition
- `/api/telegram/webhook` -- authenticated via HMAC secret instead

### Security Headers

Every response includes hardened headers set by `set_security_headers()` in the app factory:

- `X-Frame-Options: DENY` -- clickjacking prevention
- `X-Content-Type-Options: nosniff` -- MIME-sniffing prevention
- `Content-Security-Policy` -- restricts script/style/font/image sources
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` -- disables camera, mic, geolocation
- `Strict-Transport-Security` -- HSTS when `REQUIRE_HTTPS=true`

### DB Retry with Exponential Backoff

`helpers/db.py` provides `db_retry(operation, label, max_attempts=3, backoff_factor=2)`. All Supabase writes (insert, update, upsert, delete) in audits, users, and sync endpoints are wrapped in `db_retry` to survive transient network failures. Wait pattern: 1s, 2s, 4s.

### Soft-Delete Pattern

Audit entries are never hard-deleted. `DELETE /api/delete` sets a `deleted_at` timestamp instead. All read queries filter with `.is_("deleted_at", "null")` so soft-deleted rows are invisible to the application. This preserves audit trails and allows recovery.

### Optimistic Locking

Each audit record carries a `version` integer. On update, the client sends its known version; the server rejects the write with HTTP 409 if the DB version has advanced, preventing silent overwrites from concurrent edits.

### Store-Based RBAC

Non-admin users can only update or delete audit entries belonging to their assigned store. Cross-store operations return HTTP 403.

### Request Tracing

Every request is tagged with a 12-character `X-Request-ID` header and logged with method, path, status code, and duration in milliseconds.

## Quick Start

### Prerequisites

- Python 3.8 or higher
- Supabase account (free tier works)
- Windows (for .exe deployment) or any OS for development

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd Pharmacy_Arc
   ```

2. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

3. **Run setup wizard**
   ```bash
   python setup.py
   ```
   
   This will:
   - Create `.env` configuration file
   - Generate secure secret keys
   - Set up admin accounts
   - Configure database connection

4. **Start the application**
   ```bash
   python app.py
   ```
   
   The app will automatically open in your default browser at `http://127.0.0.1:5013`

## 🔐 First-Time Setup

### Database Setup

1. **Create Supabase Project**
   - Go to https://supabase.com
   - Create a new project
   - Note your project URL and anon key

2. **Run SQL Schema**
   
   In Supabase SQL Editor, run:
   
   ```sql
   -- Users table
   CREATE TABLE users (
     username TEXT PRIMARY KEY,
     password TEXT NOT NULL,
     role TEXT NOT NULL,
     store TEXT
   );
   
   -- Audits table
   CREATE TABLE audits (
     id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
     created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
     date TEXT NOT NULL,
     store TEXT NOT NULL,
     reg TEXT NOT NULL,
     staff TEXT,
     gross NUMERIC DEFAULT 0,
     net NUMERIC DEFAULT 0,
     variance NUMERIC DEFAULT 0,
     payload JSONB
   );
   
   -- Indexes for performance
   CREATE INDEX idx_audits_date ON audits(date);
   CREATE INDEX idx_audits_store ON audits(store);
   CREATE INDEX idx_audits_reg ON audits(reg);
   ```

3. **Configure Application**
   
   Run the setup wizard:
   ```bash
   python setup.py
   ```
   
   Or manually edit `.env`:
   ```bash
   SUPABASE_URL=https://your-project.supabase.co
   SUPABASE_KEY=your-anon-key-here
   ```

## 👥 User Roles

### Staff
- ✅ Create audit entries
- ❌ Cannot edit or delete
- ❌ No access to analytics
- 🔒 Locked to assigned store
- ⏳ Payouts require manager approval

### Manager
- ✅ Create, edit, delete entries
- ✅ Approve staff payouts
- ✅ Close day (lock entries)
- ❌ No access to analytics
- 🔒 Locked to assigned store (unless "All")

### Admin
- ✅ Full access to analytics
- ✅ View all stores
- ✅ Manage users
- ❌ Cannot unlock closed days

### Super Admin
- ✅ All admin capabilities
- ✅ Can unlock closed days
- ✅ Access to diagnostics

## 🔧 Configuration

### Environment Variables (.env)

```bash
# Flask Configuration
FLASK_SECRET_KEY=<generate-with-setup-wizard>
FLASK_PORT=5013
FLASK_DEBUG=false

# Supabase
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_KEY=your-anon-key

# Emergency Admin Accounts (format: username:bcrypt_hash)
EMERGENCY_ADMIN_SUPER=super:$2b$12$...
EMERGENCY_ADMIN_BASIC=admin:$2b$12$...

# Security
SESSION_TIMEOUT_MINUTES=30
MAX_LOGIN_ATTEMPTS=5
LOCKOUT_DURATION_MINUTES=15

# Logging
LOG_LEVEL=INFO
LOG_FILE=pharmacy_app.log
```

### Generating Secure Values

```bash
# Generate secret key
python security.py genkey

# Hash a password
python security.py hash "YourPassword123!"

# Verify a password hash
python security.py verify "password" "$2b$12$..."
```

## 📝 Audit Trail

Every critical operation is logged:

- **LOGIN_SUCCESS** / **LOGIN_FAILED** - User authentication
- **LOGOUT** - User logout
- **CREATE** / **UPDATE** / **DELETE** - Data modifications
- **USER_CREATE** / **USER_UPDATE** / **USER_DELETE** - User management
- **ACCESS_DENIED** - Permission violations

Each entry includes:
- Timestamp (UTC)
- Actor (username)
- Role (at time of action)
- Entity type and ID
- Before/after state (for updates)
- Success/failure
- Context (IP address, store, etc.)
- Hash chain linkage

## 🆘 Troubleshooting

### Cannot Login

1. **Check emergency accounts** in `.env`
   ```bash
   python security.py verify "password" "$2b$12$..."
   ```

2. **Check account lockout**
   - Wait 15 minutes, or
   - Restart app to clear lockouts

3. **Check logs**
   ```bash
   tail -50 pharmacy_app.log | grep LOGIN
   ```

### Configuration Errors

```bash
# Validate configuration
python -c "from config import Config; Config.startup_check()"

# Regenerate .env
python setup.py
```

### Database Connection Issues

1. **Verify credentials** in `.env`
2. **Check Supabase status**: https://status.supabase.com
3. **Test connection**:
   ```bash
   python -c "
   from config import Config
   from supabase import create_client
   Config.startup_check()
   supabase = create_client(Config.SUPABASE_URL, Config.SUPABASE_KEY)
   result = supabase.table('users').select('username').limit(1).execute()
   print('Database connection OK')
   "
   ```

### Audit Log Issues

```bash
# Verify integrity
python audit_log.py verify

# If corrupted, check for manual edits or file damage
# Audit log is append-only and tamper-evident
```

---

### Step 2: Get Supabase Credentials

You need a **Supabase project** (it's free!):

1. Go to https://supabase.com/
2. Click "Start your project"
3. Create a new project (takes ~2 minutes)
4. Once ready, go to **Settings → API**
5. Copy these two values:
   - **Project URL** (looks like: `https://xxxxx.supabase.co`)
   - **anon/public key** (long string starting with `eyJ...`)

**Keep these safe - you'll need them next!**

---

### Step 3: Configure Environment Variables

**Windows:**
```powershell
# Copy the template
copy .env.example .env

# Edit with Notepad
notepad .env
```

**macOS/Linux:**
```bash
# Copy the template
cp .env.example .env

# Edit with any text editor
nano .env
# or: code .env (if you have VS Code)
# or: open -e .env (macOS TextEdit)
```

**Required Changes in .env:**

1. **FLASK_SECRET_KEY** - Generate a random key:
   ```bash
   python -c "import secrets; print(secrets.token_hex(32))"
   ```
   Copy the output and paste it.

2. **SUPABASE_URL** - Paste your Project URL from Supabase

3. **SUPABASE_KEY** - Paste your anon/public key from Supabase

**Example .env file:**
```ini
FLASK_SECRET_KEY=a1b2c3d4e5f6...64_character_hex_string
FLASK_PORT=5013
FLASK_DEBUG=false

SUPABASE_URL=https://xxxyyyzz.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Leave other values as default for now
SESSION_TIMEOUT_MINUTES=30
MAX_LOGIN_ATTEMPTS=5
LOCKOUT_DURATION_MINUTES=15
REQUIRE_HTTPS=false
LOG_LEVEL=INFO
LOG_FILE=pharmacy_app.log
```

**Save the file!**

---

### Step 4: Set Up Database Tables

In your Supabase project:

1. Go to **SQL Editor** (left sidebar)
2. Click **New Query**
3. Copy and paste this SQL:

```sql
-- USERS TABLE
create table users (
  username text primary key,
  password text not null,
  role text not null,
  store text
);

-- AUDITS TABLE  
create table audits (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  date text not null,
  store text not null,
  reg text not null,
  staff text,
  gross numeric default 0,
  net numeric default 0,
  variance numeric default 0,
  payload jsonb
);

-- INDEXES
create index idx_audits_date on audits(date);
create index idx_audits_store on audits(store);
```

4. Click **Run** (or press F5)
5. You should see "Success. No rows returned"

## 🛑 Common Errors and How to Fix Them

### Error: "Port 5013 is already in use"

**Cause:** The app is already running in another terminal, or another program is using port 5013

**Fix (Windows):**
```powershell
# Find what's using the port
netstat -ano | findstr :5013

# Kill the process (replace <PID> with the number from previous command)
taskkill /PID <PID> /F
```

**Fix (macOS/Linux):**
```bash
# Find and kill the process using port 5013
lsof -ti:5013 | xargs kill -9
```

**Or change the port** in `.env`:
```ini
FLASK_PORT=5014
```

---

### Error: "CRITICAL ERROR: Cloud Client Init Failed"

**Cause:** Can't connect to Supabase (wrong URL or key)

**Fix:**
1. Double-check your `SUPABASE_URL` in `.env`
2. Double-check your `SUPABASE_KEY` in `.env`
3. Make sure you copied the **anon/public** key (not the service_role key)
4. Test the connection:
   ```bash
   curl https://your-project.supabase.co/rest/v1/
   ```
   Should return `{"message":"The server is running..."}`

---

### Error: "Invalid credentials" when logging in

**Cause:** Wrong username or password, or user doesn't exist in database

**Fix:**

**Option 1: Use emergency admin account**
- Username: `super`
- Password: `password`

**Option 2: Create a user manually in Supabase**
1. Go to Supabase → SQL Editor
2. Run this (generates hash for password "admin123"):
   ```sql
   INSERT INTO users (username, password, role, store) 
   VALUES ('admin', '$2b$12$LQv3c1yqBWVHxkd0LHAkCOYz6TtxMQJqhN8/LewY5NU7hlZiOXE4u', 'admin', 'All');
   ```
3. Login with `admin` / `admin123`

---

### Error: "Module not found" or "No module named 'flask'"

**Cause:** Python packages not installed

**Fix:**
```bash
pip install -r requirements.txt
```

**If still fails:**
```bash
pip3 install -r requirements.txt
```

**If STILL fails (Windows):**
```powershell
python -m pip install -r requirements.txt
```

---

### Error: Python command not found

**Cause:** Python not installed or not in PATH

**Fix (Windows):**
1. Reinstall Python from https://www.python.org/downloads/
2. **Check "Add Python to PATH"** during installation
3. Restart terminal/PowerShell

**Fix (macOS):**
```bash
# Install Python via Homebrew
brew install python3
```

**Fix (Linux):**
```bash
# Ubuntu/Debian
sudo apt update && sudo apt install python3 python3-pip

# Fedora/RHEL
sudo dnf install python3 python3-pip
```

---

### Error: Table doesn't exist (SQL error)

**Cause:** Forgot to create database tables in Supabase

**Fix:** Go back to **Step 4** above and run the SQL to create tables

---

### Error: "Account locked" after too many failed logins

**Cause:** Brute-force protection kicked in after 5 failed login attempts

**Fix:** Wait 15 minutes, or delete the lockout file:

**Windows:**
```powershell
del lockout_state.json
```

**macOS/Linux:**
```bash
rm lockout_state.json
```

Then restart the app.

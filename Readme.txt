4. OPERATIONAL WORKFLOWS
================================================================================

[A] DAILY ENTRY (The Cashier)
1. Log in as Staff.
2. Select Date, Register, and Staff Name.
3. Enter Revenue (Cash + Cards breakdown).
4. Enter Deductions (Taxes) - Button "Auto 11.5%" helps calculate this.
5. Enter Payouts (Expenses taken from drawer).
6. Enter Opening Float (default $150.00) and Actual Cash Count.
7. Click "Finalize & Upload".
   - If Payouts > $0, entry is saved as "Pending Approval".

[B] MANAGER REVIEW & CLOSING
1. Log in as Manager.
2. Go to "History" tab.
3. Filter by "Today".
4. Review entries. If "PAYOUT PENDING" badge appears, verify physical receipt 
   and click the Green Checkmark button to Approve.
5. Once all registers are entered and correct:
   - Click blue "Close Day" button.
   - Review the summary popup (Total Cash, Variance).
   - Click "Print / Save PDF" for physical records.
   - Click "Close & Lock Day".
   *RESULT:* The day is now locked. Staff cannot add new entries; 
    Managers cannot edit existing ones.

[C] ANALYTICS (The Executive)
1. Log in as Admin/Super Admin.
2. Go to "Command Center".
3. Use the "Control Bar" to filter by Store or Date Range.
   - Logic: "Last 7 Days" includes Today + previous 6 days.
   - Comparison: "vs prev" compares against the exact same number of days 
     immediately preceding the selected window.
   - Data is localized (No timezone shifting).

[D] OFFLINE MODE
1. If internet fails, the app detects the connection error.
2. Data is saved locally to a file: `offline_queue.json`.
3. Location: Usually in `C:\Users\Name\` or `OneDrive`.
4. When internet returns, a yellow "⚠️ Sync" button appears in the top header.
5. Click "Sync" to upload offline records to Supabase.

5. BUILDING THE EXECUTABLE (.EXE)
================================================================================
To compile the Python script into a standalone Windows application:

1. PREREQUISITES:
   - Python 3.x installed.
   - Libraries: `pip install flask supabase pyinstaller`
   - Assets: Ensure `app.py`, `logo.png`, and `carthage.png` are in the folder.

2. BUILD COMMAND:
   Open Terminal in the folder and run:
   
   pyinstaller --noconsole --onefile --add-data "logo.png;." --add-data "carthage.png;." app.py

3. RESULT:
   - The `app.exe` will be generated in the `dist/` folder.
   - You can create a shortcut to this .exe and place it on the Desktop.

6. TROUBLESHOOTING & ERROR CODES
================================================================================

- "Day is already CLOSED/LOCKED"
  CAUSE: A Manager has finalized the day.
  FIX: Log in as 'super' (Super Admin) and delete the "DAY_CLOSE" record 
  in the History tab (it looks like a summary row).

- "Gross mismatch"
  CAUSE: The Total Gross entered does not match (Cash + Sum of all Cards).
  FIX: Check the math on the breakdown fields.

- "Taxes look too high"
  CAUSE: Security feature. Taxes (State+City) > 20% of Cash Sales.
  FIX: Verify tax entry. If legitimate exception, an Admin must enter it.

- App closes immediately upon opening
  CAUSE: Another instance is already running on Port 5016.
  FIX: Open Task Manager -> Details -> End Task "app.exe" or "python.exe".

7. TECHNICAL SPECIFICATIONS (FOR DEVELOPERS)
================================================================================
- Framework: Flask (Microframework)
- Frontend: Template String (Jinja2) served directly.
- CSS Framework: Custom "Enterprise" CSS (Turquoise Theme).
- Validation: Double-layer (Client-side JS + Server-side Python).
- Timezone Handling: 
  - Frontend: Uses strictly local browser time (new Date(y,m,d)).
  - Backend: Stores "YYYY-MM-DD" strings to avoid UTC conversion issues.

  - Every Edit/Approval action appends to the `payload['auditTrail']` JSON array.
  - Tracks: Timestamp, User, Action, Previous Values.
